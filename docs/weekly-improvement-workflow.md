# Tikur Abay Weekly Improvement Workflow

## Weekly Cadence

### 1. Issue Triage
- Review all issues logged during the week.
- Group by severity: critical, high, medium, low.
- Group by area: operations, finance, HR, agreements, chat, uploads, access.

### 2. Fix Priority
- Prioritize anything that blocks trips, payments, signing, maintenance, or access.
- Prioritize recurring issues over one-off cosmetic issues.
- Separate pilot blockers from post-launch enhancements.

### 3. Release Planning
- Select fixes for the next release batch.
- Confirm owner, test scope, and target date.
- Keep a short release note for business users and admins.

### 4. Admin Feedback Capture
- Capture direct feedback from executive, operations, finance, HR, and marketing owners.
- Note unclear pages, duplicate info, slow workflows, and missing context.
- Feed repeated requests into the next sprint backlog.

## Weekly Output
- Updated issue list with disposition
- Prioritized fix list
- Release notes draft
- Owner assignments
- Feedback summary by role
